Create `backend/app/api/agent_admin.py` with:

**Pydantic schemas** (defined at top of file, following admin.py pattern):
- `CreateAgentRequest`: name (str), description (Optional[str]=None), system_prompt (str), model (str), max_iterations (int=5), temperature (float=0.3), allowed_tool_names (List[str]=[])
- `UpdateAgentRequest`: all fields Optional (PATCH semantics) -- name, description, system_prompt, model, max_iterations, temperature, allowed_tool_names, is_active
- `AgentAdminResponse`: all AgentConfig fields with `class Config: from_attributes = True`

**Endpoints** (all require `require_admin` dependency):
1. `GET /agents` -- list agents, filter by is_active unless `include_inactive=True` query param. Order by name.
2. `POST /agents` -- create agent. Auto-generate slug from name (lowercase, spaces to hyphens, strip non-alphanumeric except hyphens). Check for duplicate name/slug (400 if exists). Add AuditLog with entity_type="agent", action="create".
3. `GET /agents/{agent_id}` -- get single agent by ID. 404 if not found.
4. `PATCH /agents/{agent_id}` -- partial update. Build changes dict with old/new values for each changed field. If name changes, re-generate slug and check uniqueness. Add AuditLog with action="update" and changes dict. If system_prompt changed, truncate to first 100 chars in changes to avoid huge audit entries.
5. `DELETE /agents/{agent_id}` -- soft delete (set is_active=False). Add AuditLog with action="delete". Return `{"status": "deactivated", "id": agent_id}`.
6. `GET /agents/tools` -- return all tools from TOOL_REGISTRY as list of `{name, description, category, parameter_count}`. Import TOOL_REGISTRY inside function to avoid circular imports.
7. `GET /agents/usage` -- aggregate usage stats from AgentUsageLog joined with AgentConfig. Group by agent_id, agent_name. Return total_calls (COUNT), total_prompt_tokens (SUM), total_completion_tokens (SUM), total_cost_usd (SUM), avg_duration_ms (AVG). Optional start_date/end_date query params filter on AgentUsageLog.created_at.

IMPORTANT: The `/agents/tools` route MUST be defined BEFORE `/agents/{agent_id}` in the router to prevent FastAPI treating "tools" as an agent_id parameter. Same for `/agents/usage`.
