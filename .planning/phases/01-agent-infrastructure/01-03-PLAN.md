    2. `GET /{slug}` (agent detail):
       - Depends on: db session, require_member

    3. `POST /run` (run agent with SSE streaming):
       - Depends on: db session, require_member
       - Accept RunAgentRequest body
       - Look up AgentConfig by request.agent_slug, 404 if not found, 400 if not is_active
       - Build user_context: {"email": current_user.email, "role": current_user.role, "user_id": current_user.id}
       - If request.context, merge into user_context under "page_context" key
       - Create async generator that:
         a. Records start_time = time.time()
         b. Tracks usage_data and tool_count from events
         c. Iterates over run_agent_streaming(config, request.message, user_context)
         d. For each event dict, yield `ServerSentEvent(data=json.dumps(event), event="agent")`
         e. Track: if event type is "usage", capture tokens; if "tool_start", increment tool_count; if "error", record error
         f. After generator completes, create AgentUsageLog with captured data:
            - agent_id=config.id, user_id=current_user.id, model_used=config.model
            - prompt_tokens, completion_tokens from usage event
            - total_cost_usd: use `litellm.completion_cost()` if available, else 0.0
            - tool_calls_count, duration_ms=int((time.time()-start_time)*1000)
            - success=True/False based on error events
         g. Commit usage log to db
       - Return EventSourceResponse(generator())

       CRITICAL import: `from fastapi.sse import EventSourceResponse, ServerSentEvent`
       This requires FastAPI >= 0.135.0 (updated in Plan 01 requirements.txt).

       IMPORTANT: The usage log creation must happen inside the generator (after all events yielded) because the generator runs lazily. If you try to create the log outside the generator, it runs before streaming starts.

       Alternative approach if usage logging inside generator is tricky: yield a special internal "complete" event that the generator handler catches to create the log, or use a try/finally in the generator.
